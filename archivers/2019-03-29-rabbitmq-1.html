Mar 29, 2019

Rabbitmq 单节点部署笔记

环境说明

系统：Centos 6.5 erlang：>= 19.3 RabbitMQ：3.7.13

Rabbit官网地址
https://www.rabbitmq.com/
Erlang官方地址
http://www.erlang.org/

配置 YUM 源

Erlang repo

/etc/yum.repos.d/rabbitmq-erlang.repo

[rabbitmq-erlang] name=rabbitmq-erlang baseurl=http://dl.bintray.com/rabbitmq/rpm/erlang/20/el/6 gpgcheck=0 repo_gpgcheck=0 enable=1

Rabbitmq repo

/etc/yum.repos.d/rabbitmq.repo

[bintray-rabbitmq-server] name=bintray-rabbitmq-rpm baseurl=http://dl.bintray.com/rabbitmq/rpm/rabbitmq-server/v3.7.x/el/6/ gpgcheck=0 repo_gpgcheck=0 enabled=1

验证 YUM 源并执行安装

验证 Erlang

yum list | grep erlang

安装 Erlang

yum install -y erlang.x86_64

验证 Rabbitmq

yum list | grep rabbitmq

安装Rabbitmq

yum install -y rabbitmq-server.noarch

启动与登陆

启动MQ

/etc/init.d/rabbitmq-server start

启动管理插件

rabbitmq-plugins enable rabbitmq_management rabbitmq_web_stomp rabbitmq_stomp

其中web_stomp和stomp插件官网介绍如下（推荐开启）：

rabbitmq_web_stomp https://www.rabbitmq.com/web-stomp.html rabbitmq_stomp https://www.rabbitmq.com/stomp.html

登陆MQ

开启管理插件后，会看到有 15672 端口被监听

netstat -tunlp 查看端口

访问 15672 端口

http://10.100.24.51:15672/#/users

使用默认用户： guest（密码：guest）登陆

报错 User can only log in via localhost 

这个报错是因为新版本对 guest 用户做了限制，只允许从localhost（即服务器本地登陆），官方解释：https://www.rabbitmq.com/access-control.html

可以通过修改配置文件开启远程访问

也可通过新增管理用户的方式，修改登陆用户
rabbitmqctl add_user admin admin
rabbitmqctl set_user_tags admin administrator
rabbitmqctl set_permissions -p / admin '.*' '.*' '.*'
